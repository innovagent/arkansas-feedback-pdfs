# arkansas-feedback-pdfs

Discovery PDFs for the Arkansas-Tax-Rebate-Portal client feedback portal,
generated by the `notion-feedback-pipeline` Claude skill.

Each `pdfs/TRB-NNN - <submitter> - Discovery.pdf` is attached to the matching
Notion ticket via a `<pdf>` block pointing at the file's raw URL on this repo,
because GitHub serves PDFs with `Content-Type: application/pdf` and inline
disposition — which Google Drive does not, breaking Notion inline rendering.

## Why a separate public repo

- Notion's `<pdf>` block only renders external URLs that serve PDF bytes
  inline. Google Drive forces `Content-Disposition: attachment`, so Notion
  shows "Failed to load PDF file" for every Drive-hosted attempt.
- `raw.githubusercontent.com` serves arbitrary blob content with the right
  Content-Type and inline disposition. This works.

## Adding a new PDF

```
cp "/tmp/discovery/TRB-NNN - <submitter> - Discovery.pdf" pdfs/
git add pdfs/
git commit -m "Add TRB-NNN discovery PDF"
git push
```

Then in Notion, reference it via:

```
<pdf src="https://raw.githubusercontent.com/innovagent/arkansas-feedback-pdfs/main/pdfs/TRB-NNN%20-%20<submitter>%20-%20Discovery.pdf">TRB-NNN - <submitter> - Discovery.pdf</pdf>
```
(URL-encode spaces as `%20`.)
